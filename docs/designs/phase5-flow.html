What this diagram is (and isn't)
  1. Happy path only. No error branches — login failure, file rejection, duplicate confirmation, override-fired detour. Those live inline in the wireframe, not here.
  2. One session. The loop from Log out back to Log in is the outer edge; the diagram doesn't attempt to show multiple concurrent sessions or role-based variations.
  3. Persona-neutral. This is what the demo shows an executive audience. A PM's real day looks similar but starts at History or Portfolio, not Upload.
  4. Numbers match the plan doc's dashboard levels (01 / 02 / 03) for the reports, but the flow steps (Step 01 → 05) are just this diagram's own counter — don't confuse the two.
